# 🚀 Dynamo Skips App

Web dashboard for analyzing PyTorch Dynamo CPython test results and graph break reasons.

## Overview

This Streamlit application provides interactive visualization of:
- CPython test results across 38 modules (4,352+ tests)
- Graph break reasons and failure analysis
- Pass/skip rates by module
- Detailed test breakdowns

## Running CPython Tests

Generate fresh test data using the test runner script:

### Quick Start (All Tests)

Using Pixi (recommended):
```bash
pixi run test-runner
```

Or directly:
```bash
python cpython_test_runner.py
```

This will:
- Run all tests from `~/git/pytorch313/test/dynamo/cpython/3_13/`
- **Automatically save** raw output to `data/all_tests_output_<commit>_<date>.txt`
- Filename includes commit hash for version tracking
- Takes ~30+ minutes

### Run Specific Modules (Faster for Testing)

```bash
python cpython_test_runner.py --modules test_bool test_set test_dict
```

### Custom Filenames

```bash
# Save raw output to custom filename (default directory is data/)
python cpython_test_runner.py --save-output my_results.txt
```

### Reuse Previous Output (For Testing Parser)

If you already have a saved raw output file, reuse it without re-running tests:

```bash
# Using pixi
pixi run test-runner-reuse

# Or directly
python cpython_test_runner.py --reuse-output data/all_tests_output_abc123f_20260404.txt
```

### Options

| Flag | Purpose |
|------|---------|
| `--modules MODULE1 MODULE2` | Run specific test modules only |
| `--save-output FILE` | Save raw test output for later reuse |
| `--reuse-output FILE` | Skip running tests, reuse previous output |
| `--output FILE` | Custom Excel output filename |

## Running the Dashboard

```bash
pixi run streamlit run app.py
```

The app will be available at `http://localhost:8501`

## Features

### 📊 Overview Tab
- Key metrics (total tests, passing, skipped, pass rate)
- Pass vs Skip breakdown by module
- Failure percentage by module
- Summary table of all modules

### ⚠️ Graph Breaks Tab
- Top 20 graph break reasons
- Count and percentage breakdown
- Visual breakdown of most common issues

### 📋 Module Details Tab
- Select individual modules
- View detailed test results
- Per-test pass/fail/skip status with reasons

## Data Sources

The dashboard reads from:
- `data/all_tests_output_*.txt` - Full test output logs

Data is generated by running tests (see "Running CPython Tests" section above).

**Filename Format**: `all_tests_output_<11-char-commit>_<YYYYMMDD>.txt`
- Commit hash included for version tracking
- Date only (not timestamp) for cleaner filenames
